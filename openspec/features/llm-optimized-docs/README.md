# LLM优化文档功能档案

## 当前状态
- 当前版本：v1
- 状态：已完成
- 最近变更：optimize-entry-files-for-llm

## 功能背景
当前入口文件（AGENTS.md、CLAUDE.md等）主要是面向人类阅读的文档结构，包含大量解释性文字、表格和示例。这导致LLM解析效率低下、指令歧义多、结构不清晰。本功能旨在优化这些文件，使其更适合LLM解析和执行，同时保持人类可读性。

## 能力范围

### 当前支持
- 优化所有入口文件（AGENTS.md、CLAUDE.md、`.github/copilot-instructions.md`、`.codebuddy/rules/*/RULE.mdc`）
- 引入更清晰的指令结构
- 减少歧义，提高LLM解析效率
- 针对不同平台优化不同版本
- 引入新的解析机制

### 非目标
- 不改变现有技能系统的核心逻辑
- 不删除必要的规则内容
- 不破坏现有平台的兼容性
- 不引入非Markdown格式

## 关键决策
- 2026-05-16：采用渐进式优化方案，原因：风险最低，可分阶段实施，保持向后兼容
- 2026-05-16：引入结构化标记系统，原因：帮助LLM快速识别关键信息，减少解析歧义
- 2026-05-16：明确skill间调用关系，原因：用户特别强调要让不同模型都能理解skill间的调用关系
- 2026-05-16：按5点结构重组文档，原因：用户提出更清晰的指令结构需求
