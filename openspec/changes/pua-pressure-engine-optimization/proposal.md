# PUA 压力引擎优化提案

## Why
当前 PUA 流程存在"徒有其表"的问题：
1. 方法论路由没有真正应用：虽然有方法论路由表，但没有在开发过程中实时引入
2. 压力升级机制分散：每个 skill 都有"压力升级（内联保底）"章节，内容重复
3. 表面文章：PUA 技能没有真正介入开发流程
4. 缺乏统一引擎：没有统一的压力升级处理中心
5. 维护困难：修改压力升级逻辑需要修改多个文件

## What Changes
1. **强化方法论路由**：将 PUA 技能优化为方法论智能路由引擎，确保开发过程中实时引入对应方法论
2. **优化 PUA 技能**：将 `pua/SKILL.md` 优化为统一的压力升级引擎
3. **清理其他 skill**：删除其他 skill 中的压力升级废话
4. **设计介入机制**：让 PUA 技能能够真实介入开发流程
5. **统一压力升级**：所有压力升级逻辑由 PUA 技能统一处理

## Capabilities
### PUA 技能功能模块（非独立 skill）

**注意**：以下功能模块是 PUA 技能的内部功能，不是独立的 skill。它们在 `skills/pua/SKILL.md` 中统一实现。

- **方法论智能路由**：按任务类型选择最优方法论并在开发过程中实时引入，通过 `references/` 目录下的方法论文件实现
- **压力升级引擎**：统一处理所有压力升级逻辑，通过 `pua-escalation` skill 实现
- **开发过程监控**：监控开发过程中的问题，通过 PUA 行为协议实现
- **问题介入处理**：发现并介入处理问题，通过 `pua-escalation` skill 实现

### Modified Capabilities
- `pua-gate`: 门禁系统，将压力升级逻辑委托给 PUA 引擎
- `pua-escalation`: 升级器，将成为 PUA 引擎的一部分
- 所有其他 skill：删除压力升级废话，改为调用 PUA 引擎

## Impact
- 后端：影响所有使用 PUA 流程的 skill
- 数据：无持久化变更
- 依赖：无新增外部依赖

## Risk Assessment
- 变更风险等级：R0
- 影响文件数：约 20+ 个 skill 文件
- 是否命中复合升级：否
