# PUA 压力引擎优化 - 技术设计

## 1. 问题分析

### 1.1 当前问题

1. **压力升级机制分散**：每个 skill 都有"压力升级（内联保底）"章节，内容重复
2. **PUA 技能定位模糊**：主要是"味道"和"旁白"提供者，没有真正介入开发流程
3. **请求接口冗余**：PUA skill 中有请求接口和上报信息，增加了复杂度
4. **缺乏统一监控**：没有集中的问题发现和介入处理机制

### 1.2 搜索证据

通过搜索发现：
- 至少 209 处"压力升级"或"pua-escalation"匹配
- 每个 skill 都有类似的压力升级章节
- PUA 技能当前主要是"味道层"和"references 提供者"

## 2. 设计方案

### 2.1 核心架构

```
PUA 压力引擎（统一处理）
├── 方法论智能路由（核心：按任务类型选择最优方法论）
├── 压力升级引擎（统一处理所有压力升级）
├── 开发监控者（监控开发过程）
├── 问题介入中心（发现并介入处理问题）
└── 味道与旁白（保留现有功能）
```

### 2.2 PUA 技能重构

#### 2.2.1 新定位

**PUA 技能**：方法论智能路由引擎 + 统一的压力升级引擎 + 开发过程监控者 + 问题介入处理中心

#### 2.2.2 核心功能

1. **方法论智能路由（核心功能）**
   - 按任务类型自动选择最优方法论
   - **真实引导 LLM 读取对应方法论文件**
   - 开发过程中实时应用方法论指导
   - 支持失败模式智能切换

   **方法论文件路由**：
   | 任务类型 | 推荐味道 | 方法论文件 | 核心方法 |
   |---------|---------|-----------|---------|
   | Debug/修 Bug | 🔴 华为 | `references/methodology-huawei.md` | RCA 根因分析 + 蓝军自攻击 |
   | 构建新功能 | ⬛ Musk | `references/methodology-tesla.md` | The Algorithm: 质疑→删除→简化→加速→自动化 |
   | 代码审查 | ⬜ Jobs | `references/methodology-apple.md` | 减法优先 + 像素级完美 + DRI |
   | 调研/搜索 | ⚫ 百度 | `references/methodology-baidu.md` | 搜索是第一生产力 |
   | 架构决策 | 🔶 Amazon | `references/methodology-amazon.md` | Working Backwards + 6-Pager |
   | 性能优化 | 🟡 字节 | `references/methodology-bytedance.md` | A/B Test + 数据驱动 |
   | 部署/运维 | 🟠 阿里 | `references/methodology-alibaba.md` | 定目标→追过程→拿结果闭环 |
   | 任务模糊 | 🟠 阿里 | `references/methodology-alibaba.md` | 通用闭环（默认） |

   **真实引导机制**：
   - 接到任务后，先读取 `references/methodology-router.md` 确定任务类型
   - 根据任务类型，引导 LLM 读取对应的方法论文件
   - 在开发过程中，按照方法论文件的指导执行
   - 失败时，引导 LLM 读取其他方法论文件

2. **统一压力升级引擎**
   - 集中处理所有压力升级逻辑
   - 提供标准化的压力升级接口
   - 支持压力等级自动升级

3. **开发过程监控**
   - 监控开发过程中的问题
   - 检测连续失败、质量下滑等信号
   - 自动触发介入机制

4. **问题介入处理**
   - 发现问题后主动介入
   - 提供结构化的问题处理流程
   - 支持方法论智能路由

5. **味道与旁白**（保留现有功能）
   - 味道切换和旁白
   - References 宝藏地图

### 2.3 其他 Skill 优化

### 2.4 接口设计

#### 2.4.1 方法论路由接口（核心）

```markdown
## 方法论智能路由接口

PUA 技能根据任务类型真实引导 LLM 读取对应方法论文件：

1. **任务类型识别**：分析用户需求，识别任务类型
2. **方法论文件读取**：引导 LLM 读取对应的方法论文件
3. **实时应用**：在开发过程中按照方法论文件的指导执行
4. **失败切换**：失败时引导 LLM 读取其他方法论文件

使用方式：
```
use_skill("pua")
```

PUA 技能会：
1. 分析任务类型（Debug/新功能/审查/调研等）
2. 引导 LLM 读取对应方法论文件：
   - Debug/修 Bug → 读取 `references/methodology-huawei.md`
   - 构建新功能 → 读取 `references/methodology-tesla.md`
   - 代码审查 → 读取 `references/methodology-apple.md`
   - 调研/搜索 → 读取 `references/methodology-baidu.md`
   - 架构决策 → 读取 `references/methodology-amazon.md`
   - 性能优化 → 读取 `references/methodology-bytedance.md`
   - 部署/运维 → 读取 `references/methodology-alibaba.md`
3. 在开发过程中按照方法论文件的指导执行
4. 失败时引导 LLM 读取其他方法论文件
```

#### 2.4.2 压力升级接口

```markdown
## 压力升级接口

当需要压力升级时，调用 PUA 技能：

```
use_skill("pua")
```

PUA 技能会：
1. 检测当前压力等级
2. 执行相应的压力升级动作
3. 提供方法论路由
4. 输出压力旁白
```

#### 2.4.3 监控接口

```markdown
## 开发监控接口

PUA 技能会自动监控：
- 连续失败次数
- 质量下滑信号
- 跳过步骤倾向
- 空口完成行为

检测到问题时自动介入。
```
