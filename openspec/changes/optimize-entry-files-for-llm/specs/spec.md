# 优化入口文件以更适合LLM解析需求规范

## 概述
优化所有入口文件（AGENTS.md、CLAUDE.md、`.github/copilot-instructions.md`、`.codebuddy/rules/*/RULE.mdc`），使其更适合LLM解析和执行，同时保持人类可读性。重点明确skill间的互相调用关系，让不同模型都能理解。

## 功能需求

### 1. 更清晰的指令结构
- **层次化组织**：按用户提出的5点结构重组文档
  1. 角色和项目背景（20-30行）
  2. 配置中枢索引（Rules/Skills/Wiki/MCP的路径和触发场景）
  3. 七项核心职责
  4. 十阶段工作流调度指令
  5. 沟通原则和硬性约束
- **减少嵌套**：避免过深的Markdown嵌套层级
- **明确优先级**：关键规则放在前面，辅助信息放在后面

### 2. 减少歧义
- **消除模糊表述**：将"可能"、"应该"等模糊词替换为明确指令
- **统一术语**：确保同一概念使用相同术语
- **明确条件**：将隐含条件显式化

### 3. 提高LLM解析效率
- **结构化标记**：引入明确的标记系统（如 `[REQUIRED]`、`[OPTIONAL]`、`[CRITICAL]`）
- **指令优先级**：明确哪些规则必须遵守，哪些可以灵活处理
- **减少冗余**：删除重复内容，合并相似规则

### 4. 明确skill间互相调用关系
- **调用关系图**：在文档中明确skill之间的调用逻辑
- **依赖关系**：明确哪些skill依赖其他skill
- **路由逻辑**：清晰展示从入口skill到具体skill的路由路径
- **返回机制**：明确skill执行完成后如何返回调用者

### 5. 平台特定优化
- **AGENTS.md**：作为跨平台中枢，保持通用性
- **CLAUDE.md**：针对Claude Code优化，强调`use_skill`机制
- **`.github/copilot-instructions.md`**：针对GitHub Copilot优化，内联完整规则
- **`.codebuddy/rules/*/RULE.mdc`**：针对CodeBuddy优化，强调`alwaysApply`和完整规范

## 验收标准

### 功能验收
- [ ] LLM能正确解析优化后的规则，无歧义理解
- [ ] LLM能正确执行skill间调用逻辑
- [ ] 所有平台都能正常加载和使用优化后的文件

### 效率验收
- [ ] LLM解析时间减少（通过实际测试验证）
- [ ] LLM响应更准确（通过测试用例验证）
- [ ] 减少LLM需要处理的冗余信息

## 非目标
- 不改变现有技能系统的核心逻辑
- 不删除必要的规则内容
- 不破坏现有平台的兼容性
- 不引入非Markdown格式（如YAML、JSON）
- 不改变PUA G0-G4门禁基本含义

## 边界条件
- **向后兼容**：优化后的文件必须与现有技能系统兼容
- **平台差异**：不同平台的优化版本可以有差异，但核心规则必须一致
- **人类可读性**：优化后的文件仍需保持人类可读性
- **维护成本**：优化后的文件应易于维护和更新
