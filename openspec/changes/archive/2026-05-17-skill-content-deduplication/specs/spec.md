# 技能文件内容去重与职责明确化需求规范

## 概述
消除 PUA 技能套件中多个文件的重复内容，明确每个技能的核心职责，建立清晰的引用关系，优化用户引导流程。

## 功能需求

### 1. 职责明确化
- **pua/SKILL.md**：方法论路由引擎 + 味道文化（权威来源）
  - 包含：方法论路由表、味道判定协议、旁白协议、P8 思维
  - 不包含：门禁流程、入口纪律
- **pua-gate/SKILL.md**：自适应门禁（权威来源）
  - 包含：门禁判断逻辑、档位定义、风险评估、澄清协议
  - 不包含：味道判定协议详细定义、方法论路由
- **using-superpowers-pua/SKILL.md**：入口纪律 + 路由表
  - 包含：入口原则、路由表、Owner 四问、红旗信号
  - 不包含：味道判定协议详细定义、门禁流程详细定义
  - 引用：`pua` 和 `pua-gate` 作为权威来源
- **brainstorming-pua/SKILL.md**：OpenSpec 流程
  - 包含：四层渐进流程、文档模板、确认门控
  - 保持现有职责不变

### 2. 引用链机制
- 建立明确的引用关系，避免重复定义
- 引用格式：`详见 {skill-name} 技能的 {章节}`
- 确保引用的准确性和一致性

### 3. 引导流程优化
- 用户可逐步跟随技能流程
- 每个技能有清晰的入口和出口
- 避免用户在多个文件间跳转困惑

### 4. 相关文件同步更新
- **PUA-FLOW.md**：更新流程说明，反映新的职责划分
- **skill-call-diagram.md**：更新调用关系图，反映新的引用关系
- **其他引用文件**：确保所有引用都指向正确的权威来源

## 验收标准
- [ ] 四个指定文件无重复内容
- [ ] 技能职责划分明确，每个技能有清晰的核心职责
- [ ] 引用关系正确，所有引用都指向权威来源
- [ ] 引导流程清晰，用户可逐步跟随
- [ ] 相关文件同步更新，无孤立引用
- [ ] 向后兼容，现有调用关系不变

## 非目标
- 不改变现有技能的调用关系和入口
- 不增加新的技能或功能
- 不改变现有的门禁逻辑和风险评估
- 不改变现有的 OpenSpec 流程

## 边界条件
- **边界情况1**：如果某个内容在多个文件中都有定义，以哪个为准？
  - 处理方式：以职责最相关的技能文件为准，其他文件引用
- **边界情况2**：如果引用链断裂（被引用的文件不存在或内容改变）？
  - 处理方式：建立引用检查机制，确保引用的准确性
- **边界情况3**：如果用户使用旧版本的技能文件？
  - 处理方式：保持向后兼容，新内容不影响旧流程
