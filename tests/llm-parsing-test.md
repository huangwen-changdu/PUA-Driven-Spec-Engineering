# LLM 解析效率测试用例

## 概述
本文档定义了用于测试优化后文档的LLM解析效率和准确性的测试用例。

## 测试目标
1. **功能验收**：LLM能正确解析优化后的规则，无歧义理解
2. **效率验收**：LLM解析时间减少，响应更准确
3. **兼容验收**：所有平台都能正常加载和使用优化后的文件

## 测试用例

### 测试用例 1：结构化标记识别
**目标**：测试LLM能否正确识别结构化标记

**测试步骤**：
1. 提供包含 `[REQUIRED]`、`[OPTIONAL]`、`[CRITICAL]`、`[NOTE]` 标记的文档
2. 要求LLM识别并解释每个标记的含义
3. 验证LLM能否正确区分必须遵守和可选遵守的规则

**预期结果**：
- LLM能正确识别所有标记类型
- LLM能正确解释每个标记的含义
- LLM能正确区分必须遵守和可选遵守的规则

**验证方式**：
- 命令：向LLM提供测试文档，要求识别标记
- 结果：LLM正确识别所有标记
- 判定：通过/失败

### 测试用例 2：Skill调用关系理解
**目标**：测试LLM能否正确理解skill间的调用关系

**测试步骤**：
1. 提供skill调用关系文档
2. 要求LLM描述从入口到具体skill的调用路径
3. 验证LLM能否正确理解调用条件和返回机制

**预期结果**：
- LLM能正确描述调用路径
- LLM能正确理解调用条件
- LLM能正确理解返回机制

**验证方式**：
- 命令：向LLM提供调用关系文档，要求描述调用路径
- 结果：LLM正确描述调用路径
- 判定：通过/失败

### 测试用例 3：规则解析准确性
**目标**：测试LLM能否正确解析优化后的规则

**测试步骤**：
1. 提供优化后的AGENTS.md
2. 要求LLM执行一个简单任务（如"重命名函数"）
3. 验证LLM是否遵循正确的流程（门禁→执行→验证）

**预期结果**：
- LLM能正确解析规则
- LLM遵循正确的流程
- LLM输出正确的门禁微标

**验证方式**：
- 命令：向LLM提供优化后的AGENTS.md，要求执行任务
- 结果：LLM遵循正确流程
- 判定：通过/失败

### 测试用例 4：跨平台兼容性
**目标**：测试优化后的文档在不同平台上的兼容性

**测试步骤**：
1. 分别在Claude Code、GitHub Copilot、CodeBuddy上测试
2. 验证每个平台都能正确加载和使用优化后的文档
3. 检查是否有平台特定的兼容性问题

**预期结果**：
- 所有平台都能正确加载文档
- 所有平台都能正确解析规则
- 没有平台特定的兼容性问题

**验证方式**：
- 命令：在不同平台上测试优化后的文档
- 结果：所有平台都能正常工作
- 判定：通过/失败

### 测试用例 5：解析效率对比
**目标**：对比优化前后的解析效率

**测试步骤**：
1. 记录优化前LLM解析文档的时间
2. 记录优化后LLM解析文档的时间
3. 计算效率提升百分比

**预期结果**：
- 优化后解析时间减少
- 效率提升至少10%

**验证方式**：
- 命令：测量优化前后的解析时间
- 结果：优化后解析时间减少
- 判定：通过/失败

### 测试用例 6：歧义减少验证
**目标**：验证优化后的文档是否减少了歧义

**测试步骤**：
1. 提供优化前后的文档对比
2. 识别优化前存在的歧义表述
3. 验证优化后是否消除了这些歧义

**预期结果**：
- 优化后歧义表述减少
- 关键规则表述清晰无歧义

**验证方式**：
- 命令：对比优化前后的文档
- 结果：优化后歧义减少
- 判定：通过/失败
