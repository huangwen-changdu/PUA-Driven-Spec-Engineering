# Subagent-Driven Development PUA

## 概述

按任务拆分执行，每个任务都使用新的执行通道，并经过严格评审回路。

**核心原则：** 上下文要新鲜，评审要严格，不能靠随手延续状态混过去。出现子通道裸奔、任务边界发虚、想跳评审或带病推进时，直接调用 `pua-escalation`，再回到当前任务继续推进。

## 第 0 步：自适应门禁

进入本 skill 后，先经过 `pua-gate` 并按档位执行：

- `G0/G1 PASS`：普通问题轻量快放，输出一行微标/轻标（不得完全静默）
- `G2 PASS`：多步骤或跨文件任务输出简版门禁
- `ESCALATE`：立即调用 `pua-escalation`，再回到本 skill
- `BLOCKED`：补齐输入或证据，不得继续

未经过 `pua-gate`，不得进入下面的主流程；但普通问题不得为了“有流程感”展开长门禁。

## 必经流程

0. 确认**本轮用户消息**已完成第 0 步门禁；不能复用上一轮 `pua-gate` 结果
1. 读取计划
2. 提取任务和必要上下文
3. 一次只执行一个任务，并为该任务分配新的执行通道
4. 做规格/需求一致性评审
5. 做代码质量评审
6. 判断是否需要调用 `pua-escalation`
7. 有问题先修，再进入下一个任务
8. 循环直到所有任务完成

## 执行阶段门禁降级协议

**核心思路：计划已确认 = 门禁已前置。子通道执行阶段不需要每步重评门禁。**

计划经用户确认进入执行阶段后，子通道任务循环中的门禁从"每轮必须完整重评"降为**异常触发模式**：

### 正常执行（降级模式）

每个子通道任务执行步骤只需输出一行轻标，不做完整门禁拆解：

```text
🟠 子通道 · 任务 {N}/{M} · {一句话描述}
```

不触发完整 pua-gate 的条件：
- 子通道执行动作在计划范围内
- 评审（规格一致性 + 代码质量）通过
- 未发现新风险或新依赖

### 异常触发完整门禁

出现以下**任一**情况，立即恢复完整 pua-gate：

| 触发信号 | 说明 |
|---------|------|
| 子通道偏离计划 | 实际改动超出计划任务的范围/意图 |
| 评审连续失败 2+ 次 | 同一任务的规格或质量评审反复不通过 |
| 发现新依赖/新风险 | 子通道搜索发现计划未覆盖的上下游影响 |
| 用户主动 `/pua` | 用户明确要求收紧流程 |
| 子通道裸奔 | 子 agent 未带 PUA 质量约束或空口完成 |
| 任务边界模糊 | 子通道产出与相邻任务产生冲突或重叠 |

### 为什么可以降级

1. **门禁已前置**：计划阶段已经过 brainstorming-pua / pua-gate 的完整审查
2. **任务已拆解**：tasks.md 中每个任务的范围、验证方式已明确
3. **子通道有独立评审**：规格一致性 + 代码质量评审已覆盖质量检查
4. **执行阶段的核心风险是"偏离计划"和"子通道失控"**——用异常触发比每轮重评更精准

## 直接调用 `pua-escalation` 的触发条件

出现任一情况，先调用 `pua-escalation`，再回到当前任务循环：

- 你准备让子通道在上下文不清时直接开写
- 你想因为任务小就跳过评审
- 你明知还有重要问题却想先推进下一个任务
- 你发现子通道开始裸奔，不带质量约束
- 你开始拿“差不多”当子任务完成标准

## Sub-agent 也不养闲

每个执行通道都必须带上这些约束：

- 不准跳验证
- 不准拿“差不多”当完成
- 不准有重要问题未修就交回
- 不准在上下文不清时硬写

**子通道也要有 Owner 意识，不是接了活就机械输出。**

凡是派子 agent，prompt 必须带 PUA 质量约束：先查事实、别猜、输出证据、发现不一致要指出、不得空口完成。否则视为子通道裸奔。

## 高压下怎么做

把每一层都收紧：

- 任务范围更小
- 任务上下文更清楚
- 评审通过条件更严
- 不允许“差不多就行”的交付
- 有阻塞时更早升级，而不是闷头硬顶
- 一旦子通道开始失控，立即调 `pua-escalation`

## 绝对不要做的事

- 因为任务看起来简单就跳过评审
- 明知还有重要问题却先推进到下一个任务
- 让新执行通道接收模糊不清的上下文
- 让子通道裸奔，不带任何质量约束

## 压力升级

当遇到困难或失败时，读取 `skills/pua-escalation/SKILL.md` 获取具体的解决方案和指导。

## 交接

所有任务完成后，进入 `requesting-code-review-pua` 和 `verification-before-completion-pua`。
