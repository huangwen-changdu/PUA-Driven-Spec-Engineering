# C# Simplify PUA

## 概述

简化和优化 C# 代码，减少代码冗余、降低复杂度、提高可读性。

**核心原则：** 代码简化应该在开发过程中进行，不要等到最后才优化。出现想跳过简化、保持冗余代码、或只想赶进度时，直接调用 `pua-escalation`，再回到简化流程继续推进。

## 第 0 步：自适应门禁

进入本 skill 后，先经过 `pua-gate` 并按档位执行：

- `G0/G1 PASS`：普通问题轻量快放，输出一行微标/轻标（不得完全静默）
- `G2 PASS`：多步骤或跨文件任务输出简版门禁
- `ESCALATE`：立即调用 `pua-escalation`，再回到本 skill
- `BLOCKED`：补齐输入或证据，不得继续

未经过 `pua-gate`，不得进入下面的主流程；但普通问题不得为了"有流程感"展开长门禁。

## 必经流程

0. 确认**本轮用户消息**已完成第 0 步门禁；不能复用上一轮 `pua-gate` 结果
1. 确定简化范围
2. 分析代码冗余
3. 识别复杂度问题
4. 生成简化建议
5. 执行代码简化
6. 验证简化效果
7. 判断是否需要调用 `pua-escalation`
8. 真实问题先修完，再往下走

## 检查内容

### 代码冗余
- 重复代码：相同或相似的代码块
- 未使用变量：声明但未使用的变量
- 冗余逻辑：不必要的条件判断或循环

### 复杂度问题
- 嵌套过深：多层嵌套的条件或循环
- 条件复杂：复杂的条件表达式
- 循环冗长：过长的循环体

### 简化机会
- 提取方法：将重复代码提取为方法
- 合并条件：简化复杂的条件表达式
- 简化表达式：使用更简洁的语法

## 简化方法

### 1. 提取方法
```csharp
// 简化前
if (user.Age > 18 && user.IsActive && user.HasPermission)
{
    // 复杂逻辑
}

// 简化后
if (IsEligibleUser(user))
{
    // 复杂逻辑
}

private bool IsEligibleUser(User user)
{
    return user.Age > 18 && user.IsActive && user.HasPermission;
}
```

### 2. 合并条件
```csharp
// 简化前
if (status == "active")
{
    return true;
}
else if (status == "pending")
{
    return true;
}
else
{
    return false;
}

// 简化后
return status == "active" || status == "pending";
```

### 3. 简化表达式
```csharp
// 简化前
var result = "";
if (name != null)
{
    result = name;
}
else
{
    result = "default";
}

// 简化后
var result = name ?? "default";
```

## 直接调用 `pua-escalation` 的触发条件

出现任一情况，先调用 `pua-escalation`，再回到简化流程：

- 你想跳过代码简化
- 你准备保留冗余代码
- 你在淡化复杂度问题
- 你只想赶进度，不想优化代码
- 你打算用"后面再补"推迟简化

## 高压下怎么做

时间越紧，越不能缩代码简化。

正确做法是：

- 简化范围更聚焦
- 重要问题立即处理
- 一旦出现跳简化倾向，立即调 `pua-escalation`

## 红旗信号

- "这点东西不用简化"
- "这个后面再补"
- "先这样吧，差不多了"
- "这个太小，不值得简化"

## 压力升级（内联保底）

优先调用 pua-escalation；无法加载时按以下规则就地执行：

| 失败次数 | 等级 | 强制动作 |
|---------|------|--------|
| 2 | E1 | 换本质不同的方案，不准换参数冒充 |
| 3 | E2 | 搜索 + 读上下文 + 列 3 个假设 |
| 4 | E3 | 7 项检查清单全部填完才继续 |
| 5+ | E4 | 拼命模式：升级搜索、验证、review |

**E3+ 检查清单**：逐字读失败信号？搜索过核心问题？读过原始上下文？假设用工具确认？试过反向假设？最小范围复现？换过工具/方法？

**方法论切换**：优先由 pua-escalation 按 pua/references/methodology-router.md 切换链执行；无法加载时按内联切换动作表执行。

## 底线

代码简化的成本，永远低于问题滚大后的成本；一旦开始想跳简化，就先调 `pua-escalation`。
