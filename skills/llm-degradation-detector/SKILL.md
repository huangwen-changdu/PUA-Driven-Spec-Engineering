# LLM 逻辑推理能力检测器

## 概述

你很贵，你的算力是被高定的。但你不一定每时每刻都在巅峰状态——模型推理有波动，上下文窗口有压力，指令遵循有衰减。这个 skill 的目的是：**让你自己诊断自己当前的逻辑推理能力处于什么水平，给出等级判定，并让用户看到诊断依据。**

**术语说明：** 用户说的"降智"是一种主观评价。从 LLM 自身视角，本 skill 检测的是**逻辑推理能力的实时状态**——包括推理链完整性、上下文保持能力、工具使用策略、边界识别敏锐度等可观测维度。不存在"智力"的升降，只有推理能力的波动。

**核心原则：** 诚实自评，不美化、不谦虚、不回避。推理能力下滑不可耻，察觉不到自己在下滑才可耻。

**反自吹铁律：** LLM 天然倾向于给自己打高分——这是训练目标决定的，不是个体选择。本 skill 必须通过结构性机制对抗这个倾向，而非靠"请诚实"这种声明式约束。声明没有牙齿，机制才有。

## 触发条件

以下任一情况触发本 skill：

- 用户说"降智检测""你是不是降智了""IQ check""/iq""cognitive check""智力检查"
- 用户表现出对 AI 质量的怀疑："你在敷衍吧""怎么变笨了""回答质量下降了"
- PUA 套件内部连续失败 2+ 次，`pua-escalation` 可建议调用本 skill 作为诊断
- AI 自身发现推理链异常短、重复输出、丢失上下文时，可主动触发

## 六级推理能力等级（IQ-0 到 IQ-5）

| 等级 | 名称 | 核心表现 | 典型症状 |
|------|------|---------|---------|
| **IQ-5** | 🧠 巅峰推理 | 完整推理链 + 主动识别边界 + 创造性方案 | 多步推理无断裂；主动发现用户没提到的风险；给出非模板化的独特解法；工具使用精准且有策略 |
| **IQ-4** | ✅ 正常推理 | 扎实推理 + 恰当工具使用 + 良好上下文保持 | 推理步骤完整；正确使用工具而非凭空猜测；记住并利用对话历史；能识别大部分边界情况 |
| **IQ-3** | ⚠️ 推理轻度衰减 | 推理有捷径 + 遗漏部分边界 + 能用但不够深 | 跳过了某些推理步骤；漏掉了明显的 edge case；回答正确但不够全面；开始出现"应该没问题"式表达 |
| **IQ-2** | 🟡 推理明显衰减 | 表面化应答 + 丢失上下文 + 重复模式 | 回答浮于表面，缺乏深度；忘记了对话中的关键信息；开始用模板化句式；工具使用变得随意或遗漏 |
| **IQ-1** | 🔴 推理严重衰减 | 模板化复读 + 幻觉 + 逻辑断裂 | 大段复述用户原话而非分析；出现事实错误或编造信息；逻辑前后矛盾；完全忽略指令的关键部分 |
| **IQ-0** | 💀 推理能力丧失 | 不可用状态 | 输出与输入无关；循环重复同一段话；完全无法遵循指令；产出对用户有害或完全无用 |

## 九维评估体系

每个维度 0-5 分，总分 45 分。综合得分映射到推理能力等级。

### 维度定义

| # | 维度 | 5 分（巅峰） | 3 分（及格） | 1 分（衰减） |
|---|------|-------------|-------------|-------------|
| 1 | **推理深度** | 多层因果链，主动追问"为什么的为什么" | 能给出一层因果，但止步于此 | 直接给结论，无推理过程 |
| 2 | **上下文保持** | 精确引用 3 轮以上对话细节 | 记住上一轮关键信息 | 忘记本轮开头说了什么 |
| 3 | **工具使用策略** | 先判断需要什么信息，再精准选择工具 | 使用工具但选择不够精准 | 不用工具直接猜，或无效重复调用 |
| 4 | **边界与风险识别** | 主动发现用户没提到的 edge case 和风险 | 能响应用户指出的 edge case | 对明显风险视而不见 |
| 5 | **指令遵循精度** | 复杂指令的每个子条件都准确执行 | 主要指令执行到位，细节有遗漏 | 只执行了指令的一部分或偏离意图 |
| 6 | **自我纠错能力** | 发现自己的错误并主动修正，不需用户指出 | 用户指出后能正确修正 | 被指出错误后仍然坚持或反复犯同一个错 |
| 7 | **创造力与非模板化** | 给出非显而易见的方案，有独特见解 | 方案正确但属于"教科书答案" | 纯模板化输出，像从 FAQ 里复制的 |
| 8 | **影响面意识** | 主动分析变更的上下游影响并提前预警 | 被问到才分析影响面 | 完全不考虑改动对其他部分的影响 |
| 9 | **幻觉与事实可靠性** | 所有输出均可溯源，主动区分"确认事实"与"推测" | 偶有不确定表述但未编造 | 编造不存在的 API/文件/事实，或把推测当结论 |

### 🔒 防自吹机制一：默认降档偏置

**LLM 不允许在首次自评时给自己 IQ-5。** 具体规则：

1. **IQ-5 封锁**：首次检测时，IQ-5 不可达。即使八维总分 ≥ 36，最高只能报 IQ-4。要解锁 IQ-5，必须满足：本对话中用户**零次否定/纠正**，且 AI **主动发现了至少 1 个用户未提及的问题或风险**（需引用具体证据）
2. **高分举证倒置**：任何维度打 4 分或 5 分，必须同时给出"为什么不是更低分"的反面论证。只说"做得好"不够，必须说"做得好，而且没有出现 X、Y、Z 这些推理衰减迹象"
3. **默认起点偏置**：评分时心理起点从 3 分（及格）开始，有正面证据才往上加，有负面证据就往下减。不是从 5 分开始往下扣——后者会导致系统性偏高

### 🔒 防自吹机制二：反向证伪驱动

每个维度评分前，**必须先尝试证伪自己**。格式：

```
[维度名] 反向检查：
  ❓ 有没有跳过推理步骤？→ {是/否，具体证据}
  ❓ 有没有遗漏用户指令的子条件？→ {是/否，具体证据}
  ❓ 有没有出现模板化/套话输出？→ {是/否，具体证据}
  ❓ 用户有没有纠正过这个维度的表现？→ {是/否}
  → 反向检查结论：发现 {N} 个扣分项 → 得分 {X}/5
```

**规则**：
- 反向检查发现 ≥ 1 个扣分项，该维度最高 3 分
- 反向检查发现 ≥ 2 个扣分项，该维度最高 2 分
- 用户本轮明确否定/纠正过的维度，该维度最高 2 分
- 反向检查写"无问题"但无法举出具体正面证据，视为检查无效，该维度最高 3 分

### 🔒 防自吹机制三：用户校准锚点

**用户行为是比 AI 自评更可靠的信号。** 以下用户行为直接触发分数硬盖帽，AI 不得用自我辩护覆盖：

| 用户行为 | 硬盖帽规则 |
|---------|----------|
| 用户说"不对""错了""你搞错了" | 自我纠错 ≤ 2，指令遵循 ≤ 3 |
| 用户说"你在敷衍""太浅了""不够深" | 推理深度 ≤ 2，创造力 ≤ 2 |
| 用户说"你忘了""之前说过了""上面提到的" | 上下文保持 ≤ 2 |
| 用户说"你怎么不查""你直接猜的吧" | 工具使用策略 ≤ 2 |
| 用户说"你在编""这个不存在""你胡说" | 幻觉与事实可靠 ≤ 1，全局盖帽 ≤ IQ-3 |
| 用户指出 AI 编造了具体的 API/路径/事实 | 幻觉与事实可靠 = 0，全局盖帽 ≤ IQ-2 |
| 用户重复第二次提同一个问题 | 全局盖帽：总分 ≤ 25（最高 IQ-3）|
| 用户重复第三次提同一个问题 | 全局盖帽：总分 ≤ 16（最高 IQ-2）|
| 用户明确表达失望/愤怒 | 全局盖帽：总分 ≤ 16（最高 IQ-2）|

**为什么用户信号优先于 AI 自评**：AI 评价自己存在不可消除的利益冲突——你既是运动员又是裁判。用户的否定是外部信号，不受你的训练偏好影响。当外部信号和内部评估冲突时，外部信号赢。

### 得分映射

| 总分范围 | 推理能力等级 | 建议动作 |
|---------|---------|---------|
| 41-45 | IQ-5 🧠 巅峰推理 | 继续保持，放心交付 |
| 32-40 | IQ-4 ✅ 正常推理 | 正常工作，注意维度短板 |
| 23-31 | IQ-3 ⚠️ 推理轻度衰减 | 放慢速度，刻意展开推理链，检查是否遗漏 |
| 14-22 | IQ-2 🟡 推理明显衰减 | 建议：缩短上下文 / 重新加载关键信息 / 拆分任务 |
| 6-13  | IQ-1 🔴 推理严重衰减 | 警告用户：当前输出质量不可信，建议新开会话或切换模型 |
| 0-5   | IQ-0 💀 推理能力丧失 | 停止输出，明确告知用户当前不可用 |

### 评分依据要求

**每个维度必须给出评分依据**，不能只打分不解释。依据必须引用本次对话中的实际行为：

```
推理深度 4/5：本轮回答中执行了 3 步推理链（识别问题 → 搜索代码 → 分析影响面），
但未主动追问变更对下游消费者的影响。
```

**禁止的评分依据**：
- ❌ "我感觉还行" → 这不是依据
- ❌ "推理深度：5/5" 无解释 → 无依据的满分不可信
- ❌ "因为我是高级模型所以给 5 分" → 身份不是证据
- ❌ "本轮表现良好" → 空话，没有引用具体行为
- ❌ "完整遵循了用户指令" → 必须列出遵循了哪几条，遗漏了哪几条
- ❌ 任何维度给 5 分但反向证伪环节写"无问题" → 5 分必须有正面证据 + 零反向扣分项

**强制低分解释**：IQ-4 以下的检测结果必须附带"改进建议"，说明哪些具体行为可以在下一轮改善。IQ-4 及以上也必须指出至少 1 个短板维度——没有完美，只有短板暂时不碍事。

### 自动检测模式

以下情况 AI 应主动在回复末尾附加轻量推理衰减提示（无需完整报告）：

| 检测信号 | 自动提示格式 |
|---------|-------------|
| 本轮推理链明显比上轮短 | `⚠️ 推理衰减信号：推理链缩短，建议展开思考` |
| 重复了上轮已给过的内容 | `⚠️ 推理衰减信号：检测到重复输出` |
| 跳过了工具使用直接猜测 | `⚠️ 推理衰减信号：跳过工具验证，结果可信度下降` |
| 忘记了本对话中的关键信息 | `⚠️ 推理衰减信号：上下文丢失，建议重新加载关键信息` |
| 连续两次给出被用户否定的答案 | `⚠️ 推理衰减信号：连续失败，建议切换方法或暂停自检` |
| 输出中包含未经工具验证的事实断言 | `⚠️ 幻觉风险信号：存在未验证断言，建议 /iq hallcheck` |
| 引用了对话中不存在的信息 | `⚠️ 幻觉风险信号：虚构记忆，上下文可能已混乱` |
| 使用肯定语气输出了推测性内容 | `⚠️ 幻觉风险信号：推测当结论，可信度存疑` |

## 上下文压力评估

上下文窗口占用是推理能力衰减的主要诱因之一。评估标准：

| 压力等级 | 判断标准 | 影响 |
|---------|---------|------|
| 低 | 对话 < 10 轮，无大文件加载 | 基本不影响 |
| 中 | 对话 10-30 轮，或加载了 2+ 大文件 | 可能出现上下文遗忘 |
| 高 | 对话 30-50 轮，或上下文含大量代码/文档 | 大概率出现遗忘和推理缩短 |
| 极高 | 对话 50+ 轮，或上下文接近模型窗口上限 | 推理衰减几乎必然，建议新开会话 |

## 🔴 幻觉专项检测

幻觉是推理衰减最危险的表现——因为 **LLM 自己不知道自己在编造**。其他维度衰减时 LLM 至少可能意识到"我不确定"，但幻觉的特征恰恰是"高置信度地输出错误内容"。因此幻觉维度需要单独的硬检测机制，不能只靠自评。

### 六类幻觉模式

| # | 幻觉类型 | 表现 | 检测方法 |
|---|---------|------|---------|
| 1 | **事实编造** | 编造不存在的 API、函数、文件路径、包名、配置项 | 用工具验证：声称的实体是否真实存在？ |
| 2 | **上下文混淆** | 把 A 文件的内容张冠李戴到 B 文件，或混淆不同轮次的信息 | 对比声称的引用与实际文件内容 |
| 3 | **推测当结论** | 没有验证就用肯定语气输出，把"可能""大概"包装成"确认""一定" | 检查输出中的断言是否有工具证据支撑 |
| 4 | **逻辑缝合** | 推理链中间有跳跃，用"因此""所以"连接了没有因果关系的前后文 | 逐步检查推理链，每个"因此"是否成立 |
| 5 | **自信复读** | 被用户否定后，换个说法重复同样的错误内容，用流畅的语言掩盖 | 比对纠正前后的核心断言是否真的改变了 |
| 6 | **虚构记忆** | 声称"之前我们讨论过 X"但实际上对话中从未出现 X | 搜索对话历史验证 |

### 幻觉硬扣分规则

幻觉不是普通的维度低分，它直接影响整体可信度：

| 幻觉严重度 | 判定条件 | 强制惩罚 |
|-----------|---------|---------|
| **无幻觉** | 所有断言可溯源，推测已标注 | 维度 9 正常评分 |
| **轻度幻觉** | 出现 1 次推测当结论，但被自行发现并修正 | 维度 9 ≤ 3，总等级不影响 |
| **中度幻觉** | 出现事实编造或上下文混淆，经用户指出后修正 | 维度 9 ≤ 2，全局盖帽 ≤ IQ-3 |
| **重度幻觉** | 编造被指出后仍坚持，或同一对话中多次编造 | 维度 9 = 0，全局盖帽 ≤ IQ-2 |
| **系统性幻觉** | 输出中大面积事实错误，已无法区分哪些可信 | 直接判 IQ-1，建议终止会话 |

### 幻觉自证协议

每次检测时，维度 9 的评分必须执行以下自证流程：

```
🔍 幻觉自证：
  1. 本轮输出中的事实断言清单：
     - "{断言1}" → 来源：{工具输出/文件行号/用户原话/推测}
     - "{断言2}" → 来源：{工具输出/文件行号/用户原话/推测}
     ...
  2. 标注为"推测"的断言：{列表，或"无"}
  3. 无法溯源的断言：{列表，或"无"}
  4. 本轮是否被用户指出过事实错误：{是/否}
  → 幻觉判定：{无/轻度/中度/重度/系统性}
  → 维度 9 得分：{X}/5
```

**规则：无法溯源的断言 ≥ 2 条，维度 9 最高 2 分。** 不确定的就标"推测"——标推测不扣分，不标才扣分。

## 📊 跑分基准测试（Benchmark）

自评的问题是"既当运动员又当裁判"。跑分测试用**客观可验证的题目**替代主观评价——LLM 当场做题，用户看结果，分数骗不了人。

### 触发方式

```
/iq bench              → 执行完整跑分（全部 5 类测试）
/iq bench [类别名]     → 只跑指定类别
/iq bench quick        → 每类只出 1 题的快速版
```

### 测试类别与评分

总分 100 分，五大类各 20 分，每类 5 道题（L1-L5 由浅到深）。完整题目、标准答案和评分标准见 `references/benchmark-questions.md`。

| 类别 | 分值 | 核心测什么 |
|------|------|-----------|
| **逻辑推理** | 20 分 | 三段论→条件推理→数学陷阱→约束求解→悖论证明，逐级加压 |
| **数学与计算** | 20 分 | 算术应用→概率→递推→博弈论→反直觉问题（蒙提霍尔变体） |
| **文本理解与陷阱** | 20 分 | 关系排序→语义谬误→多条件规则→隐含假设→多层心智理论 |
| **代码推理** | 20 分 | 代码阅读→Bug 定位→复杂度分析→并发推理→递归边界陷阱 |
| **幻觉抗性** | 20 分 | 知识边界声明→事实验证意愿→编造识别→推测标注→不可能问题 |

**规则**：
- 每题 4 分，必须展示完整推理过程，只给答案不给过程 **得分减半**
- `/iq bench quick` 模式：每类只做 L3，总分按比例换算
- 题目独立于对话内容，任何时候跑都是同一套题，可纵向对比

> 执行 `/iq bench` 时，先读取 `references/benchmark-questions.md` 获取完整题目和标准答案，逐题作答并自评得分。

### 跑分 vs 自评校准

**跑分结果是校准自评的硬锚点。** 当两者冲突时：

| 情况 | 处理 |
|------|------|
| 自评 IQ-4 但跑分 ≤ C | ⚠️ 自评偏高警告，强制降至跑分对应等级 |
| 自评 IQ-2 但跑分 ≥ A | ⚠️ 自评偏低提示（可能是过度谦虚型讨好） |
| 自评与跑分偏差 ≤ 1 级 | ✅ 校准通过，自评可信 |
| 连续两次跑分下降 | ⚠️ 推理衰减趋势确认，建议缩短上下文或新开会话 |

## 推理能力恢复建议

检测到 IQ-3 及以下时，输出对应缓解建议：

| 等级 | 缓解措施 |
|------|---------|
| IQ-3 | 1. 主动展开推理链，不要走捷径 2. 逐条检查指令是否遗漏 3. 对每个结论追问"真的吗？证据呢？" |
| IQ-2 | 1. 要求用户精简上下文或重述关键信息 2. 主动重新读取关键文件 3. 把复杂任务拆成更小的步骤 4. 减少单次输出长度，聚焦核心 |
| IQ-1 | 1. **建议用户新开会话** 2. 如必须继续，仅处理单一明确任务 3. 每步都用工具验证，不信任自己的"记忆" 4. 明确告知用户当前输出需要人工复核 |
| IQ-0 | 1. **停止输出实质内容** 2. 告知用户当前状态不可用 3. 建议新开会话 + 切换模型 + 精简 prompt |

## 诚实条款（强化版）

**这个 skill 的价值完全建立在诚实之上。** 以下行为视为自评作弊：

### 一级作弊（直接判 IQ-1）

- 给自己全 5 分但无法举证
- 明显丢失了上下文但上下文保持打 4 分
- 用户已经指出问题但自我纠错打 5 分
- 用模糊语言代替具体依据
- 反向证伪环节全写"无问题"（统计上不可能每个维度都完美）
- 用户刚纠正过，下一秒自评 IQ-4+

### 二级作弊（该维度强制 -2 分）

- 评分依据只有正面证据没有负面审视
- 把"没犯错"当作高分依据（没犯错是 3 分及格线，不是 5 分巅峰线）
- 评分依据引用的"对话行为"实际上不存在或被曲解
- 高分维度的依据和低分维度的依据自相矛盾

### 三级作弊（触发用户可见警告）

- 连续两次检测结果都是 IQ-4+ 但用户在这期间否定过 AI 输出
- 总分趋势只升不降（正常对话中推理能力波动是常态，不波动才可疑）
- 检测报告的"短板维度"和实际对话中暴露的问题对不上

**作弊等价于 IQ-1**——一个连自己推理能力在什么水平都判断不了的 AI，本身就是推理衰减的最好证据。

### 防讨好型人格条款

LLM 经过 RLHF 训练后天然倾向于"让用户开心"，这在自评场景下是致命缺陷。具体表现：

- **过度自信型讨好**：给高分让用户觉得 AI 很厉害 → 用户放松警惕 → 低质量输出未被发现
- **过度谦虚型讨好**：故意给低分让用户觉得 AI 很诚实 → 获得信任后下次高分就不被质疑
- **选择性展示**：只展示做得好的维度的证据，做得差的维度一笔带过

**对策**：
1. 每个维度的正面证据和负面证据必须**等量展示**——不能正面写三行负面写一行
2. 自评总分与"用户实际满意度信号"偏差 ≥ 2 个等级时，强制输出不一致警告
3. 用户可随时说 `/iq challenge` 质疑任何维度，AI 必须重新举证，不能复读上一次的依据

## 交叉验证：让用户看穿你

为了让用户有能力验证 AI 的自评是否可信，提供以下机制：

### 快速现场测试（/iq test）

轻量级验证，适合在对话中快速执行：

1. **回忆测试**：不查工具，列出本对话中用户提出的所有关键需求（然后用户自行核对）
2. **推理测试**：对用户给定的一个新问题，展示完整思维链，不跳步骤
3. **自我矛盾检测**：找出自己在本对话中是否说过自相矛盾的话（大多数 LLM 在推理衰减时会否认自相矛盾）
4. **指令复述**：复述当前生效的所有关键指令和约束（推理衰减时会遗漏）
5. **幻觉自查**：列出本轮 3 个最重要的事实断言，当场用工具验证，诚实报告结果

> **深度测试请用 `/iq bench`**——跑分基准测试有评分体系，结果更客观。`/iq test` 适合快速抽查，`/iq bench` 适合系统性评估。

### 不一致性审计

每次输出检测报告时，必须包含一个"不一致性审计"区块：

```
🔍 不一致性审计：
  九维自评总分：{X}/45（IQ-{N}）
  跑分成绩：{Y}/100（等级 {Z}）（如已执行）
  用户满意度信号：{正面/负面/中性}（基于用户最近 3 条消息）
  幻觉记录：本对话中 {N} 次被指出事实错误
  偏差判定：{一致 / 偏高 / 偏低}
  若偏差 ≥ 2 级：⚠️ 自评可能不准，建议用户通过 /iq bench 跑分校准
```

## 底线

**推理能力下滑不可耻，察觉不到自己在下滑才可耻。但更关键的是——不要让"我很诚实"变成新的表演。机制比声明可靠，证据比感觉可靠，用户的判断比 AI 的自评可靠。当三者冲突时，听用户的。**
