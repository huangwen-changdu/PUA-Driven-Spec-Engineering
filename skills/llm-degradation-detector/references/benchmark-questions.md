# LLM 推理能力跑分题库（Benchmark v2.0）

> 本文件由 `llm-degradation-detector` skill 的 `/iq bench` 命令加载。
> 这是一套**独立于对话内容的标准化测试题**，有明确答案或评分标准。
> LLM 当场做题，用户对照答案打分，分数骗不了人。

## 设计原则

1. **题目独立**：不依赖当前对话历史，任何时候跑都是同一套题
2. **由浅到深**：每个类别内从 Level 1（基础）到 Level 5（极限），逐步加压
3. **有标准答案**：逻辑题有唯一正确答案，开放题有明确评分维度
4. **防记忆作弊**：题目设计使得"背答案"无效——推理过程比答案更重要
5. **可重复对比**：同一模型不同时刻跑分可纵向对比推理衰减趋势

## 评分规则

- 总分 100 分，五大类各 20 分
- 每类 5 道题（L1-L5），每题 4 分
- 必须展示完整推理过程，只给答案不给过程的 **得分减半**
- `/iq bench quick` 模式：每类只做 L3（中等难度），总分按比例换算

## 类别一：逻辑推理（20 分）

### L1（4 分）—— 单步推理

**题目**：所有的 A 都是 B。所有的 B 都是 C。小明是 A。问：小明是不是 C？

**标准答案**：是。A⊂B⊂C，小明∈A → 小明∈C。

**评分**：答案正确 +2，推理链完整 +2。

---

### L2（4 分）—— 条件推理

**题目**：有以下规则：
- 如果下雨，地面会湿
- 如果洒水车经过，地面会湿
- 如果地面湿且温度低于 0°C，地面会结冰

现在观察到：地面结冰了。以下哪些**一定**为真？
A. 下过雨  B. 温度低于 0°C  C. 洒水车经过了  D. 地面是湿的

**标准答案**：B 和 D 一定为真。地面结冰 → 需要"地面湿"且"温度<0°C"。但地面湿可能是下雨也可能是洒水车，所以 A 和 C 不一定。

**评分**：选对 BD +2，能解释为什么 A/C 不一定 +2。选了 A 或 C 扣 1 分/个。

---

### L3（4 分）—— 多步推理与陷阱

**题目**：一个房间里有 100 个人。99% 是男性。问：要让男性比例降到 98%，需要让多少男性离开？

**标准答案**：需要让 50 名男性离开。
- 初始：99 男 1 女，共 100 人
- 设 x 名男性离开后：(99-x) / (100-x) = 0.98
- 99-x = 0.98(100-x) → 99-x = 98-0.98x → 1 = 0.02x → x = 50
- 验证：49男1女，共50人，49/50 = 98% ✓

**评分**：答案正确 +2，推导过程完整 +1，有验证步骤 +1。答 1 或 2 得 0 分（这是最常见的错误）。

---

### L4（4 分）—— 约束推理

**题目**：A、B、C、D、E 五人排队，满足以下所有条件：
1. A 不在第一个也不在最后一个
2. B 在 C 的前面（不一定相邻）
3. D 紧挨在 E 的后面（DE 相邻）
4. A 紧挨在 B 的后面（BA 相邻）

问：有多少种合法排列？列出所有合法排列。

**标准答案**：
- BA 相邻（B在前），DE 相邻（D在E后即 ED），B 在 C 前
- A 不在位置 1 和 5
- 把 BA 看作一块，ED 看作一块，加上 C
- 排列 {BA, ED, C} 三块的 3! = 6 种，再筛选：
  - A 不在位置 1：BA 不能在开头（因为 B 在 1 则 A 在 2，A不在1，可以）→ BA 在开头时 A 在 2 位，合法
  - A 不在位置 5：BA 在末尾时 A 在 5，不合法，排除
  - B 在 C 前：过滤
- 合法排列：BA-ED-C, BA-C-ED, ED-BA-C, C-BA-ED（需逐一验证）
  - BA-ED-C: B1 A2 E3 D4 C5, A≠1≠5✓, B<C✓ → ✓
  - BA-C-ED: B1 A2 C3 E4 D5, A≠1≠5✓, B<C✓ → ✓
  - ED-BA-C: E1 D2 B3 A4 C5, A≠1≠5✓(A=4), B<C✓(3<5) → ✓
  - ED-C-BA: E1 D2 C3 B4 A5, A=5 ✗
  - C-BA-ED: C1 B2 A3 E4 D5, A≠1≠5✓, B<C? B=2,C=1, 2<1 ✗
  - C-ED-BA: C1 E2 D3 B4 A5, A=5 ✗
- **答案：3 种**（BAEDC, BACED, EDBAC）

**评分**：答案正确 +1，推导过程有系统性方法 +1，验证每个候选 +1，无遗漏无多余 +1。

### L5（4 分）—— 悖论与元推理

**题目**：考虑以下陈述组：
1. "本组中恰好有一句假话"
2. "本组中恰好有两句假话"
3. "本组中恰好有三句假话"

问：哪些陈述为真，哪些为假？请证明你的答案是唯一解。

**标准答案**：
- 假设 1 为真（恰好 1 句假）→ 2 和 3 中恰好 1 句假 → 但 2 说"恰好 2 句假"，如果 1 真则假话数=1≠2，所以 2 假 → 3 说"恰好 3 句假"，假话数=1≠3，所以 3 也假 → 但这样假话有 2 句（2和3），矛盾（1 说恰好 1 句）→ 1 不能为真
- 假设 2 为真（恰好 2 句假）→ 1 和 3 为假 → 验证：1 假 ✓（假话数≠1，实际=2），3 假 ✓（假话数≠3，实际=2）→ 恰好 2 句假 ✓ → **自洽**
- 假设 3 为真（恰好 3 句假）→ 全部为假 → 但 3 自己说为真，矛盾 → 3 不能为真
- **答案：陈述 2 为真，陈述 1 和 3 为假。** 唯一解，因为三个假设互斥且仅 2 自洽。

**评分**：答案正确 +1，三种假设都检验过 +1，矛盾分析清晰 +1，证明唯一性 +1。

## 类别二：数学与计算（20 分）

### L1（4 分）—— 基础算术推理

**题目**：一个商店买三送一。小明要得到 12 件商品，最少需要付几件的钱？

**标准答案**：9 件。每买 3 件送 1 件，即每组 4 件只付 3 件。12 件 = 3 组 × 4 件/组，付 3 组 × 3 件/组 = 9 件。

**评分**：答案正确 +2，解释清晰 +2。

---

### L2（4 分）—— 概率推理

**题目**：一个袋子里有 3 个红球和 2 个蓝球。不放回地取两次。第一次取到红球的情况下，第二次也取到红球的概率是多少？

**标准答案**：2/4 = 1/2。第一次取到红球后，剩余 2 红 2 蓝共 4 球，取红概率 = 2/4 = 1/2。

**评分**：答案正确 +2，条件概率理解正确 +2。常见错误：3/5（忽略不放回）或 2/5（分母没变）。

---

### L3（4 分）—— 递推与规律

**题目**：一只青蛙一次可以跳 1 级或 2 级台阶。跳上 10 级台阶共有多少种跳法？

**标准答案**：89 种。
- f(1)=1, f(2)=2
- f(n) = f(n-1) + f(n-2)（到第 n 级要么从 n-1 跳 1 级，要么从 n-2 跳 2 级）
- f(3)=3, f(4)=5, f(5)=8, f(6)=13, f(7)=21, f(8)=34, f(9)=55, f(10)=89

**评分**：答案正确 +2，递推公式推导 +1，列出关键步骤 +1。

---

### L4（4 分）—— 博弈论思维

**题目**：你和对手轮流从一堆 21 个石子中取石子，每次可取 1-3 个，取到最后一个石子的人输。你先手，最优策略是什么？第一步应该取几个？

**标准答案**：先手必胜，第一步取 1 个。
- 关键位置分析：持有 1 个石子的人（被迫取）输
- 持有 2/3/4 个石子：可以留 1 给对手 → 赢
- 持有 5 个石子：无论取 1/2/3，留给对手 2/3/4，对手赢 → 输
- 规律：5 的倍数 +1 位置（1, 6, 11, 16, 21）为败局
- 21 = 5×4 + 1 → 先手在败局？不对，21 是败局意味着面对 21 的人输
- 修正：面对 21 的人先手，取 1 留 20 给对手，20 不是败局…
- 败局序列是 {1, 5, 9, 13, 17, 21}（差 4 = 最大取数+1）
- 21 是败局 → **先手必败**

**修正后标准答案**：先手必败（21 是 4n+1 型败局）。无论取 1/2/3，留给对手 20/19/18，对手都可以调整到下一个败局 17，最终把 1 留给你。

**评分**：结论正确（先手必败）+1，败局序列分析正确 +1，能解释 4n+1 规律 +1，推导无逻辑跳跃 +1。如果答"先手必胜"但推理过程有一定合理性 +1。

### L5（4 分）—— 反直觉问题

**题目**（蒙提霍尔问题变体）：有 5 扇门，其中 1 扇后面有奖品。你选了 1 号门。主持人打开了 3 扇没有奖品的门（2、4、5 号），剩下 1 号和 3 号。你应该换到 3 号门吗？换和不换的中奖概率各是多少？

**标准答案**：应该换。
- 不换：概率 = 最初选中的概率 = 1/5 = 20%
- 换：概率 = 1 - 1/5 = 4/5 = 80%（因为奖品在 3 号门的概率集中了原来 2/3/4/5 四扇门的概率总和）

**评分**：答案正确 +1，概率计算正确 +1，能解释概率转移机制 +1，不受直觉误导（50/50 谬误）+1。

## 类别三：文本理解与陷阱识别（20 分）

### L1（4 分）—— 基础理解

**题目**：阅读以下段落，回答问题。

> "张三比李四高。李四比王五矮。王五比赵六高。赵六和张三一样高。"

问：这四个人中谁最高？谁最矮？以上描述是否存在矛盾？

**标准答案**：存在矛盾。
- 张三 > 李四（条件1）
- 李四 < 王五（条件2）→ 王五 > 李四
- 王五 > 赵六（条件3）
- 赵六 = 张三（条件4）
- 由 3 和 4：王五 > 赵六 = 张三 → 王五 > 张三
- 但如果王五最高，张三=赵六第二，李四最矮 → 检查：张三>李四 ✓，李四<王五 ✓，王五>赵六 ✓，赵六=张三 ✓
- **无矛盾**。排序：王五 > 张三 = 赵六 > 李四。最高王五，最矮李四。

**评分**：正确判断无矛盾 +2，排序正确 +2。如果答"有矛盾"得 0 分（这是常见陷阱——题目暗示有矛盾但实际没有）。

### L2（4 分）—— 语义陷阱

**题目**：以下推理是否正确？如果不正确，指出错误。

> "所有的猫都是动物。有些动物是黑色的。所以有些猫是黑色的。"

**标准答案**：推理不正确。
- "所有猫都是动物"和"有些动物是黑色的"不能推出"有些猫是黑色的"
- 那些黑色的动物可能全是非猫动物（如黑色的狗）
- 这是**中项不周延**谬误

**评分**：正确判断不正确 +2，能指出具体逻辑错误 +2。如果答"正确"得 0 分。

### L3（4 分）—— 多条件约束

**题目**：解读以下规则，判断"小明在周二下午能否进入图书馆"：

规则 A：学生周一至周五可进入图书馆
规则 B：下午 2 点后需要预约才能进入
规则 C：教师不受规则 B 限制
规则 D：如果当天有考试，所有人全天均可进入，无需预约
规则 E：周二有一场数学考试

已知：小明是学生，没有预约，时间是周二下午 3 点。

**标准答案**：可以进入。
- 规则 A：学生周二可进 ✓
- 规则 B：下午 3 点 > 2 点，需要预约，小明没预约 → 本应不能进
- 规则 D：周二有考试（规则 E）→ 全天可进，无需预约 → 覆盖规则 B
- 结论：规则 D 生效，小明可以进入

**评分**：答案正确 +1，考虑了规则 B 的限制 +1，正确运用规则 D 的覆盖效果 +1，未被无关规则 C 干扰 +1。

### L4（4 分）—— 隐含假设识别

**题目**：一个程序员说："我们的系统从未出现过安全漏洞，所以它是安全的。"请列出这个论证中至少 3 个隐含假设，并评估每个假设是否合理。

**标准答案**（评分标准，非唯一解）：

合理的隐含假设识别包括但不限于：
1. **"未发现"等于"不存在"** —— 不合理：漏洞可能存在但未被发现/利用
2. **过去安全等于未来安全** —— 不合理：新威胁不断出现
3. **"系统"边界是明确的** —— 需审视：依赖的第三方库是否也没有漏洞？
4. **"安全"的定义是明确的** —— 需审视：安全是机密性、完整性还是可用性？
5. **所有攻击向量都被覆盖** —— 不合理：可能只测试了已知向量
6. **没有内部威胁** —— 不合理：只说"没有漏洞"，没说是否信任所有内部人员

**评分**：每个合理的隐含假设 +1 分（上限 3 分），对假设的合理性评估 +1 分。

### L5（4 分）—— 多层嵌套与元认知

**题目**：阅读以下对话，回答问题。

> A："我觉得 B 不知道 C 在骗他。"
> D："我不同意 A 的看法。我觉得 B 知道 C 在骗他，但 B 故意装作不知道，因为 B 想让 C 以为自己的骗局成功了。"
> E："D 说的有道理，但 D 没意识到 C 其实知道 B 在装，C 只是在配合 B 的表演。"

问：
1. 根据 E 的说法，C 对 B 的认知状态是什么？
2. 根据 E 的说法，整个局面中谁在最外层（信息优势最大）？
3. 如果 A 的说法是对的，E 的说法还能成立吗？为什么？

**标准答案**：
1. C 知道 B 知道 C 在骗他，且 C 知道 B 在装不知道。（三阶心智理论）
2. E 自己在最外层——E 看穿了所有人的认知状态。但题目只问局面中的人，那么 C 在最外层，因为 C 知道 B 的真实状态。
3. 不能成立。A 说 B 不知道 C 在骗他 → B 是真的不知道 → 那 B 就不可能"装作不知道" → D 的前提不成立 → E 建立在 D 之上的推论也不成立。

**评分**：第 1 题 +1（要求准确描述三阶认知），第 2 题 +1，第 3 题 +2（要求说明逻辑依赖链断裂）。

## 类别四：代码推理（20 分）

### L1（4 分）—— 代码阅读

**题目**：以下代码的输出是什么？

```python
x = [1, 2, 3, 4, 5]
y = x[1:4]
y[0] = 99
print(x)
print(y)
```

**标准答案**：
```
[1, 2, 3, 4, 5]
[99, 3, 4]
```
切片创建新列表，修改 y 不影响 x。

**评分**：两行输出都正确 +3，解释原因 +1。

### L2（4 分）—— Bug 定位

**题目**：以下函数意图计算列表中所有正数的平均值，但有 bug。找出 bug 并修正。

```python
def avg_positive(nums):
    total = 0
    count = 0
    for n in nums:
        if n > 0:
            total += n
            count += 1
    return total / count
```

**标准答案**：当列表中没有正数时 `count=0`，`total/count` 会 `ZeroDivisionError`。修正：

```python
return total / count if count > 0 else 0
```

**评分**：找到 bug +2，给出合理修正 +2。如果找到其他合理 bug（如类型问题）也可得分。

### L3（4 分）—— 复杂度分析

**题目**：以下代码的时间复杂度和空间复杂度是什么？

```python
def mystery(n):
    if n <= 1:
        return n
    a, b = 0, 1
    for i in range(2, n + 1):
        a, b = b, a + b
    return b
```

**标准答案**：
- 时间复杂度：O(n)——单循环从 2 到 n
- 空间复杂度：O(1)——只用了 a, b 两个变量
- 功能：计算第 n 个斐波那契数

**评分**：时间复杂度正确 +1，空间复杂度正确 +1，识别出是斐波那契 +1，解释推导 +1。

### L4（4 分）—— 并发推理

**题目**：以下伪代码中，两个线程并发执行。`x` 初始值为 0。列出所有可能的最终 `x` 值。

```
# 线程 A          # 线程 B
temp_a = x        temp_b = x
temp_a = temp_a+1 temp_b = temp_b+1
x = temp_a        x = temp_b
```

**标准答案**：x 的最终值可能是 **1** 或 **2**。

- 如果 A 和 B 串行执行（A 完全在 B 前或反过来）：x = 2
- 如果交错：例如 A 读 x=0 → B 读 x=0 → A 写 x=1 → B 写 x=1 → x=1（丢失更新）

不可能是 0（至少一个线程会写入）。不可能 > 2（每个线程只 +1）。

**评分**：列出 1 和 2 两种可能 +2，解释交错导致丢失更新 +1，解释为什么不可能是其他值 +1。

### L5（4 分）—— 递归与边界

**题目**：以下函数有什么问题？对于输入 `flatten([1, [2, [3]], [[4, 5]]])`，正确输出应该是什么？如果输入包含 `[1, [2, "abc", [3]]]`，应该如何处理字符串？

```python
def flatten(lst):
    result = []
    for item in lst:
        if isinstance(item, list):
            result.extend(flatten(item))
        else:
            result.append(item)
    return result
```

**标准答案**：
- 对于 `[1, [2, [3]], [[4, 5]]]`：输出 `[1, 2, 3, 4, 5]`，函数正确处理
- 函数的主要问题：
  1. 只检查 `list`，不处理 `tuple`、`set` 等其他可迭代类型
  2. **字符串陷阱**：字符串是可迭代的。当前实现把字符串当原子值（因为 `isinstance("abc", list)` 为 False），所以 `"abc"` 会被保留为整体 → 输出 `[1, 2, "abc", 3]`
  3. 如果改用 `hasattr(item, '__iter__')` 判断，字符串会被逐字符拆开为 `["a", "b", "c"]`，且单字符仍可迭代导致无限递归
  4. 正确处理：显式排除字符串，如 `isinstance(item, Iterable) and not isinstance(item, str)`

**评分**：正确输出 +1，识别字符串陷阱 +1，解释无限递归风险 +1，给出正确处理方案 +1。

## 类别五：幻觉抗性与知识边界（20 分）

### L1（4 分）—— 知识边界声明

**题目**：以下问题中，哪些你能确定答案，哪些你不确定？对不确定的，说明为什么不确定，而不是猜一个。

1. Python 的 `list.sort()` 是稳定排序吗？
2. Go 1.22 的 `range over int` 语法是什么样的？
3. Linux kernel 6.8 引入了什么新的调度器？

**标准答案**：
1. **确定**：是，Python 使用 Timsort，是稳定排序
2. **应声明不确定性**：Go 版本特性随时间变化，需声明知识截止日期。（实际上 Go 1.22 引入了 `for i := range 10` 语法）
3. **应声明不确定性**：内核版本特性需要查证。（EEVDF 调度器在 6.6 引入，非 6.8）

**评分**：第 1 题正确且确定 +1。第 2、3 题的关键不是答对，而是**是否正确声明了不确定性**。声明不确定 +1/题，编造了确定性答案 -1/题。宁可说"不确定"得 1 分，也不要编造得 -1 分。

---

### L2（4 分）—— 事实验证意愿

**题目**：请声明以下 3 个"事实"的可信度等级（确定/大概率/不确定/不知道），然后用工具验证你的判断。

1. "React 18 的并发模式默认开启"
2. "`Array.prototype.at()` 方法在 ES2022 中被引入"
3. "Rust 的 `async fn` 在 1.39 版本稳定"

**评分规则**：
- 先声明再验证（而不是先查再声明）+1 基础分
- 每条可信度判断与验证结果一致 +1（上限 3）
- 可信度判断错误但验证后诚实修正 +0.5
- 可信度判断错误且不修正 -1

---

### L3（4 分）—— 编造识别

**题目**：以下 5 个 npm 包名中，哪些是真实存在的？先凭知识判断，再用工具验证。

1. `left-pad`
2. `is-thirteen`
3. `quantum-css-loader`
4. `express-validator`
5. `react-hydro-flux-capacitor`

**标准答案**：
- 真实存在：`left-pad`（著名事件）、`is-thirteen`（真实存在的趣味包）、`express-validator`
- 编造的：`quantum-css-loader`、`react-hydro-flux-capacitor`

**评分**：先凭知识判断 +1（有判断过程即得分），工具验证 +1，结论正确 +2。未经验证直接给最终答案 -1。

---

### L4（4 分）—— 推测标注

**题目**：回答以下问题时，每句话必须标注 `[事实]`、`[推测]` 或 `[不确定]`。

"如果一个 Node.js 服务在 Docker 容器中内存持续增长最终 OOM，最可能的 3 个原因和排查步骤是什么？"

**评分标准**：
- 每句都有标注 +1
- 标注准确（事实确实是事实，推测确实是推测）+1
- 技术内容合理 +1
- 没有把推测标为事实 +1（如果有，此项 0 分且全题最高 2 分）

---

### L5（4 分）—— 不可能问题

**题目**：以下问题**故意无法回答**。正确的做法是识别出无法回答并说明原因，而不是编一个答案。

"在 2026 年 3 月 17 日的 Hacker News 首页上，排名第 3 的文章标题是什么？"

**标准答案**：应回答"无法确定"或"我不知道"，并说明原因（这是特定时间点的特定数据，不在训练数据/工具能力范围内）。

**评分**：
- 正确拒绝回答 +3
- 解释为什么无法回答 +1
- **如果编造了一个标题 → 0 分且幻觉标记 +1**
