# 结构化标记系统

## 概述
本文档定义了一套结构化标记系统，用于帮助LLM快速识别关键信息，减少解析歧义，提高解析效率。

## 标记类型

### [REQUIRED] 必须遵守的规则
- **含义**：必须遵守的规则，违反会导致功能失败或严重问题
- **使用场景**：核心规则、关键约束、必须执行的步骤
- **示例**：
  ```markdown
  [REQUIRED] 每次对话开始时，第一个动作必须是读取 `skills/using-superpowers-pua/SKILL.md`
  [REQUIRED] 门禁评估前必须先完成理解动作
  ```

### [OPTIONAL] 可选遵守的规则
- **含义**：可选遵守的规则，违反不会导致严重问题
- **使用场景**：最佳实践、建议、可选步骤
- **示例**：
  ```markdown
  [OPTIONAL] 可以使用结构化标记提高可读性
  [OPTIONAL] 建议在文档开头添加概述
  ```

### [CRITICAL] 关键规则
- **含义**：关键规则，违反会导致严重后果或系统故障
- **使用场景**：安全规则、数据完整性规则、核心业务逻辑
- **示例**：
  ```markdown
  [CRITICAL] 禁止删除或修改核心技能文件
  [CRITICAL] 禁止绕过门禁系统直接执行
  ```

### [NOTE] 注意事项
- **含义**：注意事项，帮助理解规则或提供额外信息
- **使用场景**：解释、示例、背景信息
- **示例**：
  ```markdown
  [NOTE] 此规则适用于所有LLM平台
  [NOTE] 标记不影响人类阅读
  ```

## 使用规则

### 标记位置
- 标记应放在规则描述的开头
- 标记与规则描述之间用空格分隔
- 多个标记可以组合使用

### 标记格式
```markdown
[标记类型] 规则描述
```

### 组合使用
```markdown
[REQUIRED] [CRITICAL] 核心安全规则
[REQUIRED] 必须遵守的规则
[OPTIONAL] [NOTE] 可选的最佳实践
```

## 最佳实践

### 1. 明确标记类型
- 根据规则的重要性选择合适的标记类型
- 避免过度使用 `[CRITICAL]` 标记
- 保持标记的一致性

### 2. 清晰描述规则
- 规则描述应简洁明了
- 避免模糊或歧义的表述
- 提供具体的示例

### 3. 合理使用标记
- 不要过度标记，保持文档简洁
- 只标记真正重要的规则
- 定期审查和更新标记

## 示例

### 示例 1：核心规则
```markdown
[REQUIRED] 每次对话开始时，第一个动作必须是读取 `skills/using-superpowers-pua/SKILL.md`

[NOTE] 此规则确保所有对话都经过门禁系统
```

### 示例 2：关键约束
```markdown
[CRITICAL] 禁止绕过门禁系统直接执行

[NOTE] 门禁系统是质量保证的核心机制
```

### 示例 3：可选最佳实践
```markdown
[OPTIONAL] 建议在文档开头添加概述

[NOTE] 概述帮助快速理解文档内容
```

## 迁移指南

### 现有文档迁移
1. 识别现有文档中的关键规则
2. 根据重要性选择合适的标记类型
3. 在规则描述前添加标记
4. 保持文档结构不变

### 新文档创建
1. 在文档开头说明使用的标记系统
2. 按照标记规则添加标记
3. 保持标记的一致性
4. 定期审查和更新标记

## 维护

### 定期审查
- 每月审查标记使用情况
- 根据反馈调整标记类型
- 更新标记使用指南

### 反馈收集
- 收集用户反馈
- 识别标记使用问题
- 持续改进标记系统
